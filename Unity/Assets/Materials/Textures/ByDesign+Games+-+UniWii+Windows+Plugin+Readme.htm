\pard\tx220\tx720\pardeftab720\li720\fi-720\sl320\sa240\ql\qnatural
\ls3\ilvl0\cf2 {\listtext	\'95	}For any & all commercial use, please contact ByDesign Games ({\field{\*\fldinst{HYPERLINK "http://www.bydesigngames.com/contact"}}{\fldrslt http://www.bydesigngames.com/contact}})\
{\listtext	\'95	}To install: drop .DLL file plugin into your Plugins folder of your Unity scene. Call plugin from your code using the provided plugin API (see Test scene for examples).\
{\listtext	\'95	}Test Scene; example scene & package is included to showcase the software's features & plugin API.\
{\listtext	\'95	}To connect remote, press buttons 1 & 2 on the Wii remote after loading the scene; connection can take up to 10-20 seconds.\
{\listtext	\'95	}Bluetooth drivers; if you're getting unexpected behavior, use the latest bluetooth drivers (or "stack") for your Bluetooth adapter's chip maker; this can be Toshiba, Widcomm, possibly Acer (or another lesser known manufacturer). The default Microsoft stack is very unstable and can cause buggy behavior in the scene (thanks MSFT!)\
{\listtext	\'95	}Normally the device needs to be paired to your system _before_ launching the scene. If this doesn't work, try connecting to your system while the scene is executing a path; find that it helps if you've already paired in this case. Verify the wii remote device appears in your bluetooth device list (i.e. Nintendo-RVL-CNT-01 ) and is connected (not just visible, but connected).\
{\listtext	\'95	}Default "level" value; incredibly, the same wii remote responds differently to gravity in different locations; take 1 wii remote to various places and you'll have different starting angles for what should be "level" (no rotation). If you're finding this an issue, suggest writing a calibration routine that uses the plugin.\
